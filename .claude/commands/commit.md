Generate a git commit message following the Conventional Commits specification.

Rules:

- Format: <type>(<scope>): <short description>
- Types: feat, fix, docs, refactor, test, chore, perf
- Short description: imperative mood, lowercase, no period, max 72 chars
- Add a body when the change is non-trivial, explaining WHAT and WHY (not how)
- Add BREAKING CHANGE footer if applicable

Input: I will describe what was changed. Generate the commit message.
